**User Stories:**

*   **User Story ID:** US_001
    *   **Title:** Patient Registration
    *   **Description:** As a Patient, I want to view and submit the registration page, so that I can create an account on the application.
    *   **Acceptance Criteria:**
        *   The system displays the registration page with required fields.
        *   The system validates the input data before submission.
        *   The system confirms successful registration and allows the patient to log in.

*   **User Story ID:** US_002
    *   **Title:** Patient Login
    *   **Description:** As a Patient, I want to log in with my username and password, so that I can access my account and use the application's features.
    *   **Acceptance Criteria:**
        *   The system provides a login page with username and password fields.
        *   The system validates the credentials and grants access upon successful login.
        *   The system displays an error message for invalid credentials.

*   **User Story ID:** US_003
    *   **Title:** Search Doctor Information
    *   **Description:** As a Patient, I want to search for doctor's information, so that I can find a suitable doctor for my appointment.
    *   **Acceptance Criteria:**
        *   The system provides a search function for doctor's information.
        *   The system displays search results based on the input criteria.
        *   The system allows filtering search results by various parameters (e.g., specialty, location).

*   **User Story ID:** US_004
    *   **Title:** View Doctor Information
    *   **Description:** As a Patient, I want to view detailed information about a doctor, so that I can make an informed decision about booking an appointment.
    *   **Acceptance Criteria:**
        *   The system displays detailed information about the selected doctor.
        *   The system shows the doctor's qualifications, specialties, and available time slots.

*   **User Story ID:** US_005
    *   **Title:** Book Appointment Online
    *   **Description:** As a Patient, I want to book an appointment online with a diagnostic center, so that I can schedule my tests or consultations conveniently.
    *   **Acceptance Criteria:**
        *   The system provides an option to book appointments online.
        *   The system displays available time slots for the selected diagnostic center.
        *   The system confirms the appointment and calculates the waiting time.

*   **User Story ID:** US_006
    *   **Title:** View Appointment Details
    *   **Description:** As a Patient, I want to view the details of my booked appointments, so that I can keep track of my scheduled tests or consultations.
    *   **Acceptance Criteria:**
        *   The system displays a list of booked appointments.
        *   The system shows detailed information about each appointment, including date, time, and location.
        *   The system allows patients to view and manage their appointments.

*   **User Story ID:** US_007
    *   **Title:** Search Doctor Qualifications
    *   **Description:** As a Patient, I want to search for doctors based on their qualifications, so that I can find the most suitable healthcare provider for my needs.
    *   **Acceptance Criteria:**
        *   The system provides a search function for doctor qualifications.
        *   The system displays search results based on the input criteria.
        *   The system allows filtering search results by various parameters (e.g., specialty, experience).

*   **User Story ID:** US_008
    *   **Title:** View Different Doctors Online
    *   **Description:** As a Patient, I want to view different doctors online, so that I can compare their profiles and choose the best one for my appointment.
    *   **Acceptance Criteria:**
        *   The system displays a list of available doctors.
        *   The system shows detailed information about each doctor, including qualifications and specialties.
        *   The system allows patients to filter and sort doctors based on various criteria.

*   **User Story ID:** US_009
    *   **Title:** Doctor Registration
    *   **Description:** As a Doctor, I want to register online with my profile in sync with my government registration number, so that I can create an account on the application.
    *   **Acceptance Criteria:**
        *   The system provides a registration page for doctors.
        *   The system validates the input data and government registration number.
        *   The system confirms successful registration and allows the doctor to log in.

*   **User Story ID:** US_010
    *   **Title:** Doctor Login

*   **User Story ID:** US_011
    *   **Title:** Create Doctor Profile
    *   **Description:** As a Doctor, I want to create and manage my profile, so that patients can view my information and book appointments with me.
    *   **Acceptance Criteria:**

        *   The system displays the doctor's profile information to patients.

*   **User Story ID:** US_012
    *   **Title:** Manage Appointments
    *   **Description:** As a Doctor, I want to view and manage my appointments, so that I can keep track of my scheduled consultations and tests.
    *   **Acceptance Criteria:**

*   **User Story ID:** US_013
    *   **Title:** Access Test Results
    *   **Description:** As a Doctor, I want to access test results of patients from the diagnostic center, so that I can review and analyze the results for better diagnosis and treatment.
    *   **Acceptance Criteria:**

*   **User Story ID:** US_014
    *   **Title:** Cancel/Update Appointments
    *   **Description:** As a Doctor, I want to cancel or update appointments, so that I can manage my schedule effectively and accommodate changes as needed.
    *   **Acceptance Criteria:**

        *   The system sends notifications to patients about appointment changes.
        *   The system updates the appointment status in the system.

    *   **Description:** As a Diagnostic Center, I want to search for patients associated with a particular doctor, so that I can manage their records and appointments effectively.

*   **User Story ID:** US_020
    *   **Title:** View and Upload Reports
    *   **Description:** As a Diagnostic Center, I want to view and upload reports, so that doctors and patients can access test results and other relevant information.
    *   **Acceptance Criteria:**

        *   The system allows diagnostic centers to upload reports in various formats (e.g., PDF, Word).
        *   The system displays uploaded reports to authorized users (doctors and patients).
